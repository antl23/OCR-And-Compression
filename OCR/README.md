# Book-Scan OCR Pipeline

A PyTorch pipeline for reading text from noisy, grayscale book-scan images.  
**Target: ≥ 95% character accuracy** on the held-out test split.

---

## Overview

The pipeline takes multi line grayscale scan crops as input and outputs recognised text. Because the source images are synthetic (font rendered with simulated noise), ground truth labels are bootstrapped automatically from clean renderings using Tesseract. No manual annotation is required.

The model is a **CRNN** (ResNet style CNN + BiLSTM) trained with **CTC loss** on individual text line crops. At inference time, a line segmentation step splits a full image into lines before passing each one to the model.

---

## Repository Structure

| File | Purpose |
|---|---|
| `generate_labels.py` | Runs Tesseract on clean/HR images to produce `.txt` ground truth labels |
| `dataset.py` | `BookScanDataset`, vocabulary builder, and CTC collate function |
| `model.py` | CRNN architecture (ResNet CNN + BiLSTM) and greedy CTC decoder |
| `train.py` | Training loop, checkpointing, LR scheduling, and evaluation |
| `infer.py` | Single image or batch inference |

## Data Layout

The pipeline expects the following folder structure under `data/`:

```
data/
├── simulated_noisy_images_grayscale/     
├── clean_images_grayscale/               
├── clean_images_grayscale_doubleresolution/  
├── clean_images_binaryscale_lowresolution/   
└── labels/                               ← generated by generate_labels.py
```

Image filenames follow the pattern `<FontName>_<NoiseType>_<Split>.png`, where `<Split>` is one of `TR`, `VA`, `TE`, or `RE`.

## Quick Start

### 0. Install Dependencies

```bash
pip install -r requirements.txt

# Tesseract must also be installed at the OS level:
# Ubuntu:  sudo apt install tesseract-ocr
# macOS:   brew install tesseract
```

### 1. Generate Labels (one time)

```bash
python generate_labels.py \
    --data_root data/ \
    --out_dir   data/labels/ \
    --workers   8
```

This writes one `<stem>.txt` label file per (font, noise, split) combination. The script tries the **2× clean** image first, falls back to the standard clean image, and finally the binary image if Tesseract confidence is too low. Because the source images are synthetically rendered, Tesseract accuracy on clean versions is typically very high.


### 2. Train

```bash
python train.py \
    --data_root  data/ \
    --label_dir  data/labels/ \
    --run_dir    runs/run_005 \ <-- 5 is the best model
    --epochs     50 \
    --batch_size 16 \
    --lr         3e-4
```

Training stops automatically once ≥ 95% character accuracy is reached on the validation set. Progress and metrics are written to `runs/run_000/`:

```
runs/run_001/
├── best.pt        ← best checkpoint (lowest val CER)
├── last.pt        ← most recent epoch checkpoint
├── vocab.json     ← character to index mapping (required for inference)
├── log.json       ← per epoch loss and CER
└── results.json   ← final test metrics
```

By default, the `TR`, `VA`, and `RE` splits are used for training. The `TE` split is the held out test set, a configurable number of samples (default: 150) are reserved for evaluation, and the rest are folded into training.

### 4. Evaluate

```bash
python train.py \
    --data_root  data/ \
    --label_dir  data/labels/ \
    --eval_only \
    --checkpoint runs/run_005/best.pt
```

### 5. Inference

```bash
# Single image
python infer.py \
    --image      path/to/scan.png \
    --checkpoint runs/run_005/best.pt

# Whole folder → JSON output
python infer.py \
    --image_dir  path/to/folder/ \
    --checkpoint runs/run_005/best.pt \
    --out        results.json
```

At inference time, `MultiLineCRNN.forward_image()` automatically segments the input image into text lines and concatenates the decoded output with newlines.

## Architecture

### Why Line-by-Line Instead of Full Image?

Processing a full multi line image with a standard CRNN fails because the CTC constraint requires the number of CNN time steps `T` to be at least as large as the number of output characters `L`. A full image of 8 lines × ~60 chars/line = ~480 characters, but after height-pooling the CNN might produce only 34 time steps — making training impossible.

The solution is to **segment lines first**, then run the CRNN on each line individually. A single line of ~60 characters requires `T ≈ 520` steps (one per pixel column), which satisfies `T >> L` comfortably.

### Model

```
Input  [B, 1, 32, W]
         │
    ┌────▼────────────────────────────────────────┐
    │  CNN Feature Extractor                       │
    │  Conv → BN → ReLU → ResBlock × 5            │
    │  MaxPool(2,1) × 4 + Conv(2,1)               │
    │  Height collapses 32 → 1; width preserved   │
    └────────────────────┬────────────────────────┘
                         │  [B, 256, 1, W]
                         ▼
                 squeeze + permute
                         │  [B, W, 256]
                         ▼
               ┌─────────────────────┐
               │  BiLSTM × 2 layers  │
               │  hidden = 256/dir   │
               └─────────┬───────────┘
                         │  [B, W, 512]
                         ▼
               Linear → log_softmax
                         │  [T, B, vocab+1]
                         ▼
                   CTC Loss / Greedy Decode
```

The CNN uses **height-only pooling** `(2×1)` at every stage, so the width dimension (used as CTC time steps) is never compressed. Line crops are resized to a fixed height of 32 px with their aspect ratio preserved.

### Line Segmentation

`segment_lines()` in `model.py` uses a horizontal projection profile on the binarised image to find row ranges containing text:

1. Denoise and binarise with Otsu thresholding (inverted: text = white).
2. Apply morphological closing along the horizontal axis only to merge broken characters within a line without merging adjacent lines.
3. Compute the horizontal projection profile and threshold it to find text bands.
4. If an expected line count is provided, search over a range of thresholds and optionally split merged bands at their weakest internal row until the count matches.

## Training Details

| Setting | Default |
|---|---|
| Line height | 32 px |
| Max line width | 1024 px |
| LSTM hidden size | 256 (per direction) |
| LSTM layers | 2 |
| Dropout | 0.5 |
| Optimiser | AdamW, weight decay 1e-3 |
| LR schedule | Linear warmup (5 epochs) → cosine decay |
| Loss | CTC (blank index = 0) |
| Gradient clipping | max norm 2.0 |

### Data Augmentation (train split only)

- Random brightness and contrast jitter (±50%)
- Small random rotation (±2°)
- Gaussian blur
- Random horizontal stretch (±15%)
- Additive Gaussian noise

### Legibility Filtering

The dataset performs two-stage filtering to discard crops that are unreadable (slivers, blank margins, ink smudges, letter-tip fragments at the bottom of a page). Thresholds are configurable in `dataset.py` via `_MIN_CROP_HEIGHT`, `_MIN_INK_RATIO`, `_MAX_INK_RATIO`, and `_MIN_STD_DEV`.

---

## Hyperparameter Tuning

| Symptom | Suggestion |
|---|---|
| Under-fitting | Increase `--lstm_hidden 384` or add more ResBlocks |
| Over-fitting | Lower `--lr`, increase weight decay, add more augmentation |
| Slow convergence | Use `--batch_size 32` on multi-GPU |
| Low label quality | Re-run `generate_labels.py` and inspect random samples manually |
| Width truncation | Increase `--max_w 1280` |
| Low alignment rate | Check segmentation with `segmenter.py` output; tune projection thresholds in `model.py` |
